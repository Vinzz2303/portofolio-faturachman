# TING AI 2.0 Philosophy

## Core Identity

Ting AI 2.0 is an AI Financial Intelligence Layer.

It is not:

- a trading dashboard
- a Google Finance clone
- a signal provider

## Product Principle

Less UI, more meaning.

The product should help users understand what matters before showing them more surfaces, cards, or data detail.

## Primary Focus

Ting AI 2.0 should stay centered on:

- market context
- risk awareness
- narratives
- portfolio insight

## Data Role

Data is supporting context, not the center of the experience.

Charts, metrics, and market cards may exist, but they should not become the product's main identity.

## UX Direction

Insight should appear before data.

The user should first understand:

- what is happening
- why it matters
- what risk is building
- how it may affect the portfolio

Only after that should supporting data be shown.

## Explicit Guardrails

Ting AI 2.0 must not:

- become a signal provider
- present buy/sell behavior as the core interaction
- encourage action-first flows without context
- turn into a finance portal clone
- overload the interface with charts before insight is clear

## Build Rule For 2.0

Before adding any new UI block, ask:

1. Does this improve understanding?
2. Does this strengthen market context, risk awareness, narratives, or portfolio insight?
3. Is this supporting the intelligence layer, or distracting from it?

If the answer is unclear, do not add the UI yet.
